# RAVINE: Reasoning and Vision Interrogation Evaluator

This directory contains a standalone script to evaluate Vision Language Models (VLMs) on spatial reasoning datasets, particularly those generated by VQASynth. The script is inspired by the systematic evaluation approaches found in frameworks like `LLaMA-Factory`.

## Purpose

The primary goal of `VQASynth` is to generate high-quality data. To validate the effectiveness of this data, we need a reproducible way to measure the performance of models trained on it. This evaluation script provides a minimal, testable framework to:

1.  Load a VLM fine-tuned on spatial reasoning tasks.
2.  Run inference on a benchmark dataset (e.g., a hold-out set from a `VQASynth` run).
3.  Parse the model's structured output (`<think>`/`<answer>`).
4.  Calculate and report performance metrics (e.g., accuracy).

## Usage

### 1. Installation

The script requires `transformers`, `datasets`, and their dependencies. It's recommended to install them in your project's virtual environment. 4-bit quantization is used to fit larger models, which requires `bitsandbytes` and `accelerate`.

```bash
pip install torch transformers datasets accelerate bitsandbytes Pillow
```

### 2. Authentication

You will need to be logged into the Hugging Face Hub to download the model and dataset.

```bash
huggingface-cli login
```

### 3. Running the Evaluation

Execute the `process_evaluation.py` script with the model and dataset you wish to evaluate.

**Example:** Evaluating `SpaceThinker-Qwen2.5VL-3B` on the `OpenSpaces_MC_R1` test set.

```bash
python experiments/ravine_evaluator/process_evaluation.py \
    --model_id "remyxai/SpaceThinker-Qwen2.5VL-3B" \
    --dataset_id "remyxai/OpenSpaces_MC_R1" \
    --dataset_split "test" \
    --output_dir "./eval_results" \
    --sample_size 100
```

### Arguments

-   `--model_id`: (Required) The Hugging Face Hub ID of the model to evaluate.
-   `--dataset_id`: (Required) The Hugging Face Hub ID of the dataset to use for evaluation.
-   `--dataset_split`: (Optional) The dataset split to use. Defaults to `test`.
-   `--output_dir`: (Optional) The directory where results and metrics will be saved. Defaults to `./eval_results`.
-   `--sample_size`: (Optional) Limit the evaluation to a specific number of samples. If not provided, the entire split is used.

### 4. Outputs

The script will generate two files in the specified output directory:

-   `results.jsonl`: A JSONL file where each line contains the detailed results for a single sample (question, ground truth, full model response, parsed answer, and correctness).
-   `metrics.json`: A JSON file summarizing the overall performance, including accuracy.
