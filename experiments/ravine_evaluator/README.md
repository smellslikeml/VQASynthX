# Ravine-style Scene Graph Evaluator

This script provides a minimal, experimental framework for evaluating the spatial reasoning consistency of models on VQA datasets, such as those generated by VQASynth. It is inspired by compositional evaluation benchmarks and leverages PyTorch Geometric (PyG) for data structuring.

## Principle

Instead of relying solely on matching generated text answers, this evaluator:
1.  Parses structured metadata (e.g., object bounding boxes) from a VQA dataset sample.
2.  Constructs a basic scene representation where objects are nodes and their 2D coordinates/dimensions are node features, using `torch_geometric.data.Data`.
3.  Parses the natural language question to understand the spatial relationship being queried (e.g., "left of", "taller than").
4.  Programmatically determines the "ground truth" answer by performing geometric calculations on the scene representation (e.g., comparing the coordinates of the object nodes).
5.  Compares this programmatic ground truth to the model's generated answer to calculate a "Geometric Consistency Score".

This approach tests whether a model's answer is geometrically consistent with the scene it's describing, providing a more robust metric for spatial reasoning capability.

## Usage

### 1. Installation

Ensure you have the required libraries installed. The main new dependency is PyTorch Geometric.

```bash
# Install core dependencies
pip install datasets torch

# Install PyTorch Geometric (see official docs for specific CUDA versions)
# https://pytorch-geometric.readthedocs.io/en/latest/install/installation.html
pip install torch_geometric
```

### 2. Run Evaluation

Run the script from the root of the `VQASynth` repository, pointing it to a Hugging Face dataset generated by the pipeline.

```bash
python experiments/ravine_evaluator/process_evaluation.py --dataset_name remyxai/OpenSpaces_MC_R1 --num_samples 200
```

### Command-line Arguments

- `--dataset_name`: The name of the Hugging Face dataset to evaluate (e.g., `remyxai/OpenSpaces_MC_R1`).
- `--num_samples`: The number of samples from the dataset to process.

## Limitations & Future Work

This is a proof-of-concept and has several simplifications:
- **Question Parsing & Entity Linking:** The logic to identify the relationship and the objects from the question is very basic and fragile. A robust implementation would require more advanced NLP techniques to link noun phrases (e.g., "the red forklift") to specific detected objects.
- **Graph Structure:** The graph is currently just a collection of nodes. Future work could involve creating explicit edges for relationships (adjacency, containment) and using GNNs to reason over the graph.
- **2D Only:** Evaluation is based on 2D bounding boxes. Integrating the 3D point cloud data from the VQASynth pipeline would enable much more accurate and complex spatial evaluations.
