# Ravine Evaluator

This tool provides a way to evaluate the geometric consistency of datasets generated by the `VQASynth` pipeline. It is inspired by the data validation and cleaning scripts found in the `ExplainVulD` project, which filters graph data based on structural properties to ensure quality.

Here, instead of validating graph structure, we validate the logical consistency of the generated spatial reasoning data.

## Description

The `process_evaluation.py` script performs the following steps:

1.  **Loads a VQASynth-generated dataset**. This dataset must have been created with extra metadata, specifically the 3D coordinates of the objects mentioned in the question.
2.  For each VQA sample related to distance, it **parses the model's natural language answer** to extract a numerical distance and its unit (e.g., "2.5 meters", "5 feet").
3.  It **calculates the true Euclidean distance** between the objects using their stored 3D coordinates from the scene reconstruction pipeline.
4.  It **compares the stated distance and the calculated distance**. If the relative error is within a specified tolerance, the sample is marked as "consistent".
5.  Finally, it **outputs a summary report** and a detailed JSON file logging the status of each sample.

This allows for a quantitative measure of the quality and reliability of the synthetic data.

## Input Dataset Format

The script expects a Hugging Face dataset saved to disk, containing at least the following columns for distance-based questions:

-   `answer` (string): The full answer from the VLM, e.g., "The man is about 3 feet from the pallet."
-   `obj1_3d_coords` (list of floats): The `[x, y, z]` coordinates of the first object's centroid.
-   `obj2_3d_coords` (list of floats): The `[x, y, z]` coordinates of the second object's centroid.

## Usage

1.  Ensure you have the necessary libraries installed:
    ```bash
    pip install datasets numpy tqdm
    ```

2.  Run the evaluation script on your generated dataset:
    ```bash
    python experiments/ravine_evaluator/process_evaluation.py \
      --dataset_path /path/to/your/vqasynth_dataset \
      --output_file results.json \
      --tolerance 0.5
    ```

    -   `--dataset_path`: Path to the directory containing the saved HF dataset.
    -   `--output_file`: (Optional) Name for the JSON output file.
    -   `--tolerance`: (Optional) The maximum allowed relative error to be considered consistent. Default is `0.5` (50%).

The script will print a summary to the console and save the detailed results to the specified output file.
